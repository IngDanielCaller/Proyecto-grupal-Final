# **GIT**
## **¿Qué es GIT?**
Git es un sistema de control de versiones distribuido que permite a los desarrolladores rastrear y gestionar cambios en el código fuente de proyectos, especialmente cuando trabajan en equipo. Funciona tomando "instantáneas" o "confirmaciones" del estado de los archivos, las cuales están vinculadas para crear un historial completo del desarrollo. Sus principales beneficios son la capacidad de colaborar sin conexión a internet, revertir cambios fácilmente y gestionar el código de manera eficiente a través de ramas. Fue creado por Linus Torvalds en 2005 para gestionar el código fuente del núcleo de Linux.

## ****¿Por qué usar Git?**

Git ofrece múltiples ventajas en el desarrollo y gestión de proyectos, ya que permite conservar un registro organizado de los cambios, facilitar el trabajo en equipo y experimentar sin perder información.

| Razon   | Explicacion   |
|----------|--------------|
|Historial completo de cambios |	Permite revisar y restaurar versiones anteriores.|
| Recuperación fácil	| Si algo falla, puedes volver atrás.
 |Trabajo en equipo | 	Varias personas pueden colaborar sin interferir entre sí.|
| Experimentación  segura |	Puedes probar nuevas ideas en ramas separadas.|
 |Integración con plataformas |	Funciona con GitHub, GitLab o Bitbucket para compartir proyectos.|

 ## **¿Cuándo usar Git?**

 Git puede utilizarse en cualquier proyecto que evolucione con el tiempo o requiera un registro de versiones, ya sea individual o en grupo.
 
 | Situacion  |  Ejemplo |
 |-----------|----------|
 |Desarrollo de software	|Controlar versiones de código en distintos lenguajes.|
|Trabajo académico|	Mantener diferentes versiones de informes o tesis.|
| Diseño o contenido|	Seguir cambios en archivos de texto, imágenes o gráficos.|
| Proyectos grupales|	Coordinar el trabajo de varios integrantes de un equipo.|
